# 数据信息的表示

# 不同编码之间的转化

# 转化为二进制

  1. 整数的转化：对二取余，高位在下，即在书写二进制表示时，余数从低往高写
  2. 小数的转化：每次都进行乘 2 的操作，若乘数大于 1 则保留 1 位，将小数继续计算，根据要求的精度判断计算次数

# 二进制转化为十六进制

  1. 整数部分：从右往左，每四位转化为一位十六进制数，高位补零
  2. 小数部分：从左往右，每四位转化为一位十六进制数，低位补零

# 数值数据的表示

# 补码的特性

  1. 机器零唯一，+0==-0;
  2. 符号位可以直接参与运算
  3. 减法可以变成加法
  4. 负数比正数多 1 个

# 浮点数的表示

# 浮点数的作用和原理

阶符表述阶码数值部分的正负

数符表示整个数的正负

# 浮点数的规格化

原码的规格化

  1. 正数都是 0.1 格式
  2. 负数都是 1.1 格式

尾数的最高位为 0 的话会丢失精度

左规就是为了使尾数的最高位不为 0

右规使尾数的运算结果不溢出，302.131 改为 3.02131

采用双符号位，当出现溢出时可以挽救，最高位的符号位为准

用补码表示的尾数

正数表示 符号位为 0, 数值位的最高位必须为 1

负数表示 符号位为 1, 数值位的最高位必须为 0

尾数的最高数值位必须与符号位相反

# IEEE 754

一般来说，移码 = 真值 + 偏置值 偏置值 = 2 的 n-1 次方

在该标准中 移码 = 真值 + 偏置值 偏置值 = 2 的 n-1 次方 - 1

考察 IEEE 754 二进制和十进制之间的转换

# IEEE 754 标准的转化

  1. 十进制浮点数转为 IEEE 754: 先把十进制数转化为二进制 -> 对二进制数进行左归 / 右归，使其出现 2 的次方 -> 求出数符 S,E=e+127,M=0.....
  2. IEEE 754 转化为二进制浮点数：先观察 S,E,M-> 第一位为符号位，接下来的后八位为数值位，再后面的二十三位是小数部分
